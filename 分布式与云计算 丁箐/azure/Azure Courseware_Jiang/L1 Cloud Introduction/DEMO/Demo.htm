Classic ASP and FTP - Hello ASP Web Site

Overview

In this demo, you will create a Windows Azure Web Site, and get an ASP Legacy page running in the cloud using an FTP client.

Goals

In this demo, you will see:

  1. How easy it is to create a new Web Site in Windows Azure.

  2. How to get an existing ASP Web Site working in just a few minutes by uploading it with an FTP client.

Key Technologies

  • Windows Azure Web Sites

Demo

This demo is composed of the following segments:

  1. Create a New Web Site Hosted in Windows Azure.
  2. Upload an ASP page to the Windows Azure Web Site using an FTP.
  3. Set the ASP page as the Web Site Default Document.

Create a New Web site Hosted in Windows Azure

Speaking Point

During this demo we are going to create a Windows Azure Web Site.

Lets start by opening Internet Explorer and accessing the Windows Azure Management Portal.

  1. Open Internet Explorer, go to the Windows Azure Management portal and sign in using your Microsoft Account credentials associated with your subscription.

    Speaking Point

    Thanks to the Quick Create option from the Windows Azure Management Portal, we can now get a Web Site working in the cloud in just seconds.

  2. Click Compute, Web Site and then Quick Create on the command bar. Provide an available URL (e.g. hello-world-asp) for the new web site and click Create Web Site.

    Creating a new Web Site using Quick Create

  3. Once the Web Site is created, click the link under the URL column to make sure it is working.

    Going to the new Web Site

    Web site running

Upload an ASP page to the Windows Azure Web Site using an FTP

Speaking Point

Next, we are going to get a basic ASP page working in the cloud, uploading it with an FTP.

Lets go to the dashboard panel of the Web Site we have just created.

  1. Back in the Windows Azure Management Portal, click your Web Site name, under the Name column to access the Dashboard.

    Accessing the Web Site Dashboard

    Speaking Point

    On the Dashboard we can see useful information about the Web Site such as access reports, hardware usage and database access.

    We also can find multiple options for uploading content to the Web Sites, including GIT, TFS and FTP.

  2. Show the information displayed on the Web Site Dashboard.

    Accessing the Web Site Dashboard

    Speaking Point

    To access the Web Site with an FTP, we need to specify the deployment credentials we will be using.

  3. Click Reset deployment credentials in the Quick Glance section.

    Resetting Deployment Credentials

  4. Enter a User Name and Password for the FTP.

    Setting up FTP Credentials

    Speaking Point

    For this demo we are going to access the host with the File Explorer FTP connection, however you may use the FTP client of your preference.

  5. Click the URL under FTP Hostname

    Opening Web Site FTP

  6. Insert the FTP credentials previously specified.

    Inserting FTP Credentials

  7. Once in the FTP browser, click Alt and click View | Open FTP Site in File Explorer.

    Opening FTP in File Explorer

  8. If prompted for credentials, insert them again to access using File Explorer.

    File Explorer FTP Credentials

  9. Open /site/wwwroot/ folder.

    Speaking Point

    Here we can see the Web Site content.

    Lets create a new basic ASP page.

  10. Open notepad and write the following code:

    HTML
    <%
Response.Write "<html><body>"
Response.Write "Hello world!"
Response.Write "</body></html>"
%>

  11. Name the file Hello.asp, save it to Desktop, and move it inside the FTP folder.

    Hello.asp file inside the FTP

  12. Switch to Internet Explorer and go to your Web Site URL. Then add /hello.asp at the end of the root URL.

    Hello.asp on the Cloud

Set the ASP page as the Web Site Default Document

Speaking Point

We are now going to set up the hello.asp page as the Default Web Site page, so that when we go to the root URL our page is displayed.

  1. Switch to the Windows Azure Management Portal and click Configure from the Web Site top menu.

    Web Site Configure Options

  2. Scroll down to the Default Documents section and add hello.asp as a new default document.

    Hello.asp as the Primary Default Document

  3. Click Up Arrow to make hello.asp page the default document before hostingstart.html.

    Hello.asp as the Primary Default Document.png

  4. Click Save in the bottom menu to save the changes made.

    Save Default Documents List

  5. Go to the Web Site root URL and confirm that Hello.asp is the page displayed.

    Hello.asp as the Default Page

Summary

In this demo, you saw how to deploy your Web Site to the cloud with Windows Azure, and how easy it is to upload your own Web Site using an FTP client.
